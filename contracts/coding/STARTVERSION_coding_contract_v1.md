# STARTVERSION Coding Contract v1

## Standards
- Keep changes minimal and scoped to the task.
- Prefer readable code over clever code.
- No dead files, placeholders, or commented-out blocks in final patch.

## Safety
- Preserve backward compatibility unless explicitly approved.
- Add/adjust tests or checks for behavior changes.
- Fail fast with clear error messages.

## Review minimum
Each PR must show:
- what changed
- why it changed
- how it was validated
